# 숨지마 프로젝트에 기여하기 (Contributing to Sumjima)

'숨지마'는 모든 개발자의 참여를 환영하는 오픈소스 프로젝트입니다. 여러분의 작은 기여가 모여 수수료 없는 투명한 직거래 문화를 만드는 데 큰 힘이 됩니다. 버그 수정, 새로운 기능 제안, 코드 개선 등 어떤 형태의 기여든 환영합니다!

## 🚀 어떻게 시작할까요?

기여를 시작하는 가장 좋은 방법은 GitHub 저장소의 [이슈(Issues)](https://github.com/Boksepyunsal/sumjima/issues) 목록을 살펴보는 것입니다. 특히 `good first issue` 라벨이 붙은 이슈는 프로젝트에 처음 기여하시는 분들께 적합합니다.

## 💻 개발 및 기여 과정

프로젝트는 Next.js, TypeScript, Supabase, Tailwind CSS를 기반으로 합니다. 기여 과정은 다음과 같습니다.

1.  이 저장소를 **Fork** 한 후, 로컬 환경으로 **Clone** 합니다.
2.  `README.md` 파일을 참고하여 개발 환경을 설정합니다.
3.  작업할 내용에 맞는 새 브랜치를 생성합니다. (예: `feat/new-feature`, `fix/login-bug`)
4.  코드를 수정하고, 커밋하기 전에 린트 검사를 실행합니다. (`npm run lint`)
5.  수정 사항에 대한 **Pull Request(PR)**를 생성합니다. PR에는 어떤 문제를 해결했는지, 어떻게 변경했는지 명확하게 작성해주세요.

## 📝 코드 스타일 및 원칙 (Code Style & Principles)

프로젝트의 일관성과 품질을 유지하기 위해 다음 규칙을 지켜주세요.

### 커밋 메시지 (Commit Messages)

모든 커밋 메시지는 **[Conventional Commits](https://www.conventionalcommits.org/ko/v1.0.0/)** 규칙을 따릅니다. 이를 통해 변경 내역을 명확하게 추적하고, 버전 관리를 자동화할 수 있습니다.

- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 변경
- **style**: 코드 스타일 변경 (포매팅, 세미콜론 등)
- **refactor**: 기능 변경 없는 코드 리팩토링
- **test**: 테스트 코드 추가 또는 수정
- **chore**: 빌드 프로세스나 보조 도구 변경

예시: `feat: Add user profile page`

### 코딩 스타일 (Coding Style)

TypeScript 코드는 **Airbnb TypeScript/JavaScript Style Guide**를 기반으로 작성합니다. `npm run lint` 명령어를 통해 코드 스타일을 검사하고 수정할 수 있습니다.

### 개발 원칙 (Development Principles)

우리는 더 나은 코드를 위해 다음과 같은 원칙을 지향합니다.

- **YAGNI (You Ain't Gonna Need It)**: 지금 당장 필요하지 않은 기능은 만들지 않습니다.
- **DRY (Don't Repeat Yourself)**: 코드 중복을 최소화합니다.
- **SOLID**: 객체 지향 설계의 5가지 기본 원칙을 준수합니다.
- **KISS (Keep It Simple, Stupid)**: 코드를 최대한 단순하고 이해하기 쉽게 작성합니다.

- **버그 수정**: 발견된 버그를 해결합니다.
- **새로운 기능**: `Supabase Realtime`을 이용한 실시간 채팅, 지역 기반 필터링 고도화 등 새로운 기능을 제안하고 개발합니다.
- **코드 개선**: 코드의 가독성을 높이거나 성능을 개선합니다.
- **문서 개선**: `README.md`나 이 문서를 더 명확하고 이해하기 쉽게 수정합니다.
